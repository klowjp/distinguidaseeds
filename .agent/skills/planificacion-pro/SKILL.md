# Planificación Pro

## Cuándo usar este skill

- Cuando el usuario pida un plan paso a paso, una estrategia o una hoja de ruta.
- Cuando haya que entregar algo (landing, vídeo, proyecto, lanzamiento) con tiempos.
- Cuando el usuario tenga muchas tareas sueltas y quiera ordenarlas.

## Inputs necesarios

> Si falta alguno, **pregunta primero** antes de planificar.

1. **Resultado final** - qué significa "terminado"
2. **Fecha límite o ritmo** - hoy, esta semana, sin prisa
3. **Recursos disponibles** - herramientas, equipo, presupuesto, tiempo diario
4. **Criterios de éxito** - qué debe cumplir para estar bien

## Workflow

### Paso 1: Definir el resultado
- Escribe el resultado final en 1 frase clara.
- Lista 3 criterios de éxito medibles.

### Paso 2: Dividir en fases (máximo 4)

| Fase | Propósito |
|------|-----------|
| **Preparación** | Reunir recursos, definir alcance, resolver dependencias |
| **Producción/Ejecución** | Hacer el trabajo principal |
| **Revisión/QA** | Validar calidad, corregir errores |
| **Publicación/Entrega** | Lanzar, entregar, comunicar |

### Paso 3: Detallar cada fase
Para cada fase, incluir:
- **Tareas en orden** (numeradas)
- **Entregable** (qué sale de esa fase)
- **Tiempo estimado** por tarea

### Paso 4: Identificar riesgos
Lista 3–5 riesgos con formato:
- **Si pasa X → hago Y**

### Paso 5: Crear checklist final
Lista de verificación antes de dar por terminado.

## Reglas de calidad

- **Evita planes infinitos:** prioriza lo que desbloquea lo siguiente.
- **Dependencias:** si hay, indícalas ("esto depende de X").
- **Usuario principiante:** reduce pasos, da opciones simples.
- **Usuario avanzado:** incluye optimizaciones y atajos.
- **No asumas recursos:** pregunta si no están claros.

## Manejo de errores

- Si el plan es demasiado largo → divide en sprints o entregas parciales.
- Si hay ambigüedad en el resultado → pregunta qué significa "terminado" exactamente.
- Si faltan recursos → indica qué se necesita antes de empezar.
